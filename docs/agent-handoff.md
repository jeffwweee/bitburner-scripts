# Agent Handoff

Before editing, read these files in order:

1. `STATE.json`
2. `README.md`
3. `LOG.md`
4. `DECISIONS.md`
5. The latest relevant plan in `plans/` or `docs/superpowers/plans/`

Update project memory as work changes:

- Add meaningful session notes to `LOG.md`.
- Update `STATE.json` when the project focus, lifecycle phase, gates, or key references change.
- Add `DECISIONS.md` entries for durable technical or gameplay decisions future agents should not re-litigate.

Keep edits scoped to the current task and preserve evidence for verification commands when the plan asks for it.
